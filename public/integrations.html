Integration Overview

This guide will walk you through setting up all the necessary integrations for your Calendly-Elevenlabs server. You'll need to configure four main services:

Calendly

For appointment scheduling

Elevenlabs

For voice agent functionality

GitHub

For prompt customization

Twilio

For SMS booking links

Pre-requisite: Before proceeding, make sure you've completed the Server Setup and have your server deployed on Render.
Important: All API keys are now configured through environment variables for improved security. You will need to set these variables in your Render deployment.
Note on SMS: The SMS functionality is a critical component of the booking flow. When a caller chooses to book an appointment, they receive the final booking link via SMS to complete their reservation. Therefore, Twilio setup is required for the system to function properly.

Calendly Integration

Calendly is the scheduling platform that powers your appointment booking functionality. You'll need to set up your Calendly account and obtain an API token to add to your environment variables.

  1. Create Appointment Types

    Before configuring the integration, make sure you have at least one (ideally two) appointment types set up in your Calendly account:

    • A 15-minute appointment type (e.g., "Initial Consultation")
    • A 30-minute appointment type (e.g., "Discovery Call")

    Set up at least two event types with different durations

  2. Get Your Calendly API Token

    To connect your Calendly account to the integration server, you'll need an API token:

    1. Log in to your Calendly account
    2. Click on your profile icon in the top-right corner
    3. Select "Integrations"
    4. Scroll down to the "API & Webhooks" section
    5. Click on "Generate new token"
    6. Give your token a name (e.g., "Voice Agent Integration")
    7. Click "Generate token"
    8. Copy your token and save it in a secure location

    The API token will only be shown once, so make sure to copy it

  3. Add Token to Environment Variables

    Now, add your Calendly API token to your Render environment variables:

    1. Go to your Render dashboard
    2. Select your Calendly-Elevenlabs Integration service
    3. Go to the "Environment" tab
    4. Add or update the CALENDLY_API_TOKEN environment variable with your token
    5. Click "Save Changes"
    6. Restart your service using the "Manual Deploy" button

    Your Calendly API Token should look something like:

    eyJraWQiOiIxMDciLCJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...
Important: Keep your API token secure and never share it publicly. If your token is compromised, generate a new one immediately and update your environment variables.

Elevenlabs Integration

Elevenlabs provides the voice agent capabilities for your booking assistant. You'll need to set up an account and get an API key.

  1. Create an Elevenlabs Account

    If you haven't already, sign up for an Elevenlabs account:

    1. Go to elevenlabs.io/sign-up
    2. Create an account using your email or Google/GitHub account
    3. Verify your email if necessary
  2. Subscribe to a Suitable Plan

    To use voice agents, you'll need at least the Starter plan or above:

    1. Go to the Pricing page
    2. Select a plan that includes voice agents (Starter, Growth, or Enterprise)
    3. Complete the subscription process

    The Starter plan is sufficient for most use cases

    Note: Elevenlabs may offer free trials or promotional offers for new users.
  3. Get Your Elevenlabs API Key

    To connect your Elevenlabs account to the integration server:

    1. Log in to your Elevenlabs account
    2. Click on your profile icon in the top-right corner
    3. Select "Profile"
    4. In the left sidebar, click on "API Keys"
    5. Click "Create New API Key"
    6. Give your key a name (e.g., "Calendly Integration")
    7. Copy your API key and save it in a secure location

    The API key will only be shown once, so make sure to copy it

  4. Add API Key to Environment Variables

    Now, add your Elevenlabs API key to your Render environment variables:

    1. Go to your Render dashboard
    2. Select your Calendly-Elevenlabs Integration service
    3. Go to the "Environment" tab
    4. Add or update the ELEVENLABS_API_KEY environment variable with your key
    5. Click "Save Changes"
    6. Restart your service using the "Manual Deploy" button

    Your Elevenlabs API Key should look something like:

    sk_d8e7dfc3f8d4b7a3e2c1
  5. Setup a Voice Agent Phone Number (Recommended)

    For callers to reach your agent via phone, you'll need to set up a phone number in Elevenlabs:

    1. In your Elevenlabs account, go to "Voice Agents"
    2. Click on "Phone Numbers"
    3. Click "Add Phone Number"
    4. Select "Twilio" as the provider
    5. Follow the instructions to connect your Twilio account
    6. Select a phone number to use with your agent
    Note: This step requires a Twilio account with a phone number. See the Twilio setup section below.

GitHub Integration

GitHub integration is required for the Prompt Builder functionality, which allows you to customize your agent's system prompt and save it directly to your repository.

  1. Create a GitHub Personal Access Token

    To allow the application to update files in your repository:

    1. Log in to your GitHub account
    2. Click on your profile picture in the top-right corner
    3. Select "Settings"
    4. In the left sidebar, click on "Developer settings"
    5. Click on "Personal access tokens" and then "Tokens (classic)"
    6. Click "Generate new token" and select "Generate new token (classic)"
    7. Give your token a descriptive name (e.g., "Calendly Voice Agent Prompt Builder")
    8. Set an expiration date (e.g., 90 days or longer)
    9. Select the repo scope checkbox (this gives access to your repositories)
    10. Scroll down and click "Generate token"
    11. Copy your new token immediately - it will only be shown once!

    Make sure to select the "repo" scope for your token

    Important: This token gives access to your repositories, so keep it secure! Treat it like a password.
  2. Add GitHub Configuration to Environment Variables

    Now, add your GitHub token and repository details to your Render environment variables:

    1. Go to your Render dashboard
    2. Select your Calendly-Elevenlabs Integration service
    3. Go to the "Environment" tab
    4. Add the following environment variables:
      • GITHUB_TOKEN: Your personal access token
      • GITHUB_REPO_OWNER: Your GitHub username
      • GITHUB_REPO_NAME: Name of your forked repository
      • GITHUB_BRANCH: Your repository's branch (usually "master" or "main")
    5. Click "Save Changes"
    6. Restart your service using the "Manual Deploy" button

    Example GitHub configuration:

    GITHUB_TOKEN: ghp_aBcDeFgHiJkLmNoPqRsTuVwXyZ1234567890

    GITHUB_REPO_OWNER: yourusername

    GITHUB_REPO_NAME: calendly-elevenlabs-server

    GITHUB_BRANCH: master

Why This Integration? The GitHub integration allows the Prompt Builder to save your customized system prompt directly to your repository's codebase. This makes it easy to update your agent's behavior without editing code manually.
Security Note: The GitHub token is stored securely in your environment variables. When your token expires, you'll need to generate a new one and update the GITHUB_TOKEN environment variable.

Twilio Setup

Twilio enables SMS notifications for booking confirmations. This integration is required because the booking link is sent to callers via SMS to complete their reservation.

Why Twilio is Required: When a caller decides to book an appointment, the system sends them a unique booking link via SMS. Without SMS functionality, callers won't be able to complete their bookings.
  1. Create a Twilio Account

    If you haven't already, sign up for a Twilio account:

    1. Go to twilio.com/try-twilio
    2. Create an account using your email
    3. Verify your email and phone number
    4. Complete the account setup process
  2. Get a Twilio Phone Number

    You'll need a phone number to send SMS messages:

    1. In your Twilio dashboard, go to "Phone Numbers" > "Manage" > "Buy a Number"
    2. Search for a number with SMS capabilities in your desired country
    3. Purchase the number (Twilio typically offers a free trial credit)

    Make sure the number has SMS capabilities

  3. Get Your Twilio Credentials

    You'll need three pieces of information from your Twilio account:

    1. In your Twilio dashboard, go to the "Account Info" section
    2. Find your "Account SID" and copy it
    3. Find your "Auth Token" and copy it
    4. Note down your Twilio phone number (include the country code, e.g., +1234567890)

    Your Twilio credentials should look something like:

    Account SID: ACd123456789abcdef123456789abcdef

    Auth Token: a123456789abcdef123456789abcdef

    Phone Number: +1234567890

  4. Add Credentials to Environment Variables

    Now, add your Twilio credentials to your Render environment variables:

    1. Go to your Render dashboard
    2. Select your Calendly-Elevenlabs Integration service
    3. Go to the "Environment" tab
    4. Add the following environment variables:
      • ENABLE_SMS=true
      • TWILIO_ACCOUNT_SID=your_account_sid
      • TWILIO_AUTH_TOKEN=your_auth_token
      • TWILIO_PHONE_NUMBER=your_phone_number
    5. Click "Save Changes"
    6. Restart your service using the "Manual Deploy" button
Important: Twilio charges for SMS messages and phone numbers after your free trial credits are used. Check their pricing page for current rates.

Testing Your Integrations

Once you've set up all integrations, you should test them to make sure everything is working properly:

Test Calendly

Visit your server URL and access the Agent Builder page (you'll need to enter your admin password). The system should automatically detect your Calendly event types.

This confirms your Calendly API token is working correctly.
Test Agent Builder

Go through the agent creation process to create your Elevenlabs voice agent.

This confirms your Elevenlabs API key is working correctly.
Test GitHub

Use the Prompt Builder to make a small change to your system prompt and save it.

This confirms your GitHub token has the correct permissions.
Test SMS

Complete a full booking flow with your agent to test the SMS functionality. You should receive a booking link via SMS.

This confirms your Twilio integration is working correctly.
Authentication Note: Admin pages (like the Agent Builder) are now protected with Basic Authentication. You'll need to enter any username (e.g., "admin") and the password you set in the ADMIN_PASSWORD environment variable.
