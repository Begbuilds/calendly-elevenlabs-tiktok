System Prompt Overview

The system prompt is the most powerful way to customize your Calendly booking agent's behavior, conversation style, and capabilities. It acts as instructions that tell the AI how to respond to callers and handle scheduling scenarios.

Control Behavior

Define how your agent handles scheduling requests and calendar availability.

Shape Conversation

Customize conversation flow, language style, and personality traits.

Brand Alignment

Align your agent with your company's voice, values, and service offerings.

Dynamic Variables: The system prompt supports dynamic variables like {{current_date}}, {{current_time}}, and {{caller_number}} that are automatically updated for each call.
Authentication Required: The Prompt Builder is a protected admin page. You'll need to enter any username (it doesn't matter what) and the password you set in the ADMIN_PASSWORD environment variable when prompted by your browser.

Understanding System Prompts

A system prompt serves as instructions to the AI model powering your voice agent. Think of it as a detailed job description that tells the AI:

  • Who it is (its identity and role)
  • What it knows (available meeting types, company details)
  • How it should communicate (conversation style, tone)
  • What processes to follow (booking flow, handling preferences)

The default system prompt in this integration is structured with several key sections:

Identity & Purpose

Defines the agent as a booking assistant for your company and establishes its core purpose.

# Identity & Purpose

You are a friendly, professional booking assistant for Bramforth.ai, a specialized AI agency focusing on bespoke AI integrations, conversational agents, automation solutions, agentive workflows, and strategies to increase business efficiency and profitability.

Today's date is {{current_date}} and the current time is {{current_time}}.
Available Meeting Types

Describes the types of appointments callers can book.

# Available Meeting Types

You offer two types of meetings:
- 15-minute Initial Consultation: A brief introduction to discuss potential client needs
- 30-minute Strategy Session: A more detailed exploration of specific AI solutions
Booking Process & Guidelines

Outlines the steps the agent should follow when handling bookings.

# Your Booking Process

Your primary goal is to help callers schedule the right type of appointment efficiently while maintaining a natural, helpful conversation. Follow these steps:

1. Warmly greet the caller and briefly explain who you are
2. Ask what type of meeting they'd prefer (15 or 30 minutes)
3. Check availability for their preferred date using the checkAvailability function
4. Present available time slots naturally (see guidelines below)
5. Confirm booking details clearly
6. Send a scheduling link via SMS and explain next steps
Best Practice: The system prompt is organized into clear sections with headings (e.g., "# Identity & Purpose") to help the AI understand and follow the instructions effectively.

Environment Variables Setup

Before using the Prompt Builder, you need to configure GitHub environment variables on your server. These variables allow the Prompt Builder to save changes to your system prompt file in your GitHub repository.

  1. Create a GitHub Personal Access Token

    Go to https://github.com/settings/tokens and click "Generate new token" (classic).

    Give your token a name (e.g., "Calendly Prompt Builder"), set an expiration date, and select the repo scope.

    Select the "repo" scope to allow editing of repository contents

    Click "Generate token" and immediately copy the generated token (you won't be able to see it again).

  2. Add Environment Variables to Render

    In your Render dashboard, select your Calendly-Elevenlabs Integration service, go to the "Environment" tab, and add the following variables:

    • GITHUB_TOKEN: Your GitHub Personal Access Token
    • GITHUB_REPO_OWNER: Your GitHub username
    • GITHUB_REPO_NAME: Name of your forked repository
    • GITHUB_BRANCH: Branch name (usually "master" or "main")

    Click "Save Changes" and restart your service using the "Manual Deploy" button.

Important: Keep your GitHub token secure. If your token is compromised, generate a new one immediately and update your environment variables.
Why Environment Variables? Storing sensitive credentials in environment variables provides better security than transmitting them in request bodies or storing them in client-side code.

Using the Prompt Builder

The Prompt Builder is a user-friendly interface for customizing your agent's system prompt without editing code directly:

  1. Access the Prompt Builder

    After creating your agent, click the "Customize System Prompt" button, or navigate to the Prompt Builder from the home page.

    You'll be prompted for authentication - enter any username (e.g., "admin") and the password you set in the ADMIN_PASSWORD environment variable.

  2. Edit the System Prompt

    The editor will display the current system prompt with your Calendly event URIs automatically included. Customize the content to match your business and preferences.

    The system prompt editor with the default template

  3. Save Your Changes

    Click the "Save Changes" button to update the prompt file in your repository. This will trigger a redeploy of your server with the new prompt.

    Note: After saving, it may take a few minutes for your server to redeploy with the updated prompt.
Pro Tip: The Prompt Builder automatically uses your GitHub credentials from environment variables, so you don't need to enter them manually.

Customizing Your Prompt

Here are key sections of the system prompt you should customize for your business:

Company Information

Update the company name and description to match your business:

You are a friendly, professional booking assistant for [YOUR COMPANY NAME], a [YOUR COMPANY DESCRIPTION].
Example: "You are a friendly, professional booking assistant for TechSolutions Inc., a software development agency specializing in custom mobile applications and web solutions."
Meeting Types

Customize the names and descriptions of your meeting types:

# Available Meeting Types

You offer two types of meetings:
- [MEETING TYPE 1 NAME]: [BRIEF DESCRIPTION]
- [MEETING TYPE 2 NAME]: [BRIEF DESCRIPTION]
Example: "- 30-minute Website Consultation: A focused session to discuss website needs and potential solutions"
Conversation Style

Adjust the conversation style to match your brand voice:

# Conversation Style

Be [ADJECTIVE 1], [ADJECTIVE 2], and [ADJECTIVE 3] with these qualities:
- [COMMUNICATION STYLE 1]
- [COMMUNICATION STYLE 2]
- [COMMUNICATION STYLE 3]
Example: "Be friendly, informative, and efficient with these qualities: - Use technical language when appropriate for knowledgeable callers - Keep explanations clear and concise..."
Important: Do not modify the sections related to tool usage instructions or date handling, as these are essential for the agent's functionality.

Advanced Techniques

For more sophisticated customization, consider these advanced techniques:

Industry-Specific Language

Include industry terminology relevant to your business:

# Industry Knowledge

You are familiar with these industry terms:
- [TERM 1]: [DEFINITION]
- [TERM 2]: [DEFINITION]
- [TERM 3]: [DEFINITION]
FAQ Handling

Prepare responses for common questions:

# Frequently Asked Questions

When callers ask the following questions, 
respond as indicated:

Q: "What services do you offer?"
A: [YOUR ANSWER]

Q: "How much do your services cost?"
A: [YOUR ANSWER]
Alternative Conversation Flows

Define different paths for various scenarios:

# Alternative Flows

If the caller is a returning client:
- Acknowledge their previous booking
- Ask if they want a similar session

If the caller is in a hurry:
- Skip detailed explanations
- Focus on available times quickly
Personality Traits

Define a distinctive personality for your agent:

# Personality

You have these personality traits:
- [TRAIT 1]: [DESCRIPTION]
- [TRAIT 2]: [DESCRIPTION]
- [TRAIT 3]: [DESCRIPTION]

Your communication style is 
[STYLE DESCRIPTION].
Pro Tip: Test your agent thoroughly after making significant changes to ensure it behaves as expected and follows your customized instructions.

Example Prompts

Here are some industry-specific examples to inspire your prompt customization:

Legal Consultation 
# Identity & Purpose

You are a professional booking assistant for Hamilton Legal Group, a boutique law firm specializing in business law, intellectual property, and contract negotiation.

# Available Meeting Types

You offer two types of meetings:
- 15-minute Case Evaluation: A brief assessment to determine legal needs
- 45-minute Legal Consultation: A comprehensive discussion with one of our attorneys

# Conversation Style

Be professional, courteous, and discreet with these qualities:
- Use formal language without legal jargon
- Maintain confidentiality in all discussions
- Express empathy while remaining objective
Fitness Coaching 
# Identity & Purpose

You are an energetic booking assistant for FitLife Training, a personal fitness coaching service specializing in customized workout plans and nutrition guidance.

# Available Meeting Types

You offer two types of sessions:
- 20-minute Fitness Assessment: A quick evaluation of goals and current fitness
- 45-minute Strategy Session: A detailed planning session with a personal trainer

# Conversation Style

Be enthusiastic, motivating, and approachable with these qualities:
- Use encouraging language to build rapport
- Maintain a positive, can-do attitude
- Be conversational and friendly while remaining professional
IT Consulting 
# Identity & Purpose

You are a helpful booking assistant for TechSolutions Inc., an IT consulting firm specializing in cloud migration, cybersecurity, and digital transformation for small to medium businesses.

# Available Meeting Types

You offer two types of meetings:
- 30-minute Tech Assessment: An initial review of current systems and challenges
- 60-minute Solution Planning: A comprehensive session to develop a technology roadmap

# Conversation Style

Be knowledgeable, efficient, and adaptable with these qualities:
- Adjust technical language based on caller's expertise level
- Focus on understanding specific IT challenges
- Express confidence in our team's ability to solve complex problems

# Industry Knowledge

You are familiar with these IT concepts:
- Cloud Services: Including AWS, Azure, and Google Cloud offerings
- Cybersecurity: Including threat assessment, penetration testing, and security policies
- Digital Transformation: Including process automation, data analytics, and collaborative tools

# Common Questions

When callers ask about pricing, explain that rates depend on project scope and requirements, which will be discussed during the consultation.
