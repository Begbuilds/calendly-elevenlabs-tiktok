Getting Started

This guide will walk you through the complete setup process for your own Calendly-Elevenlabs Integration server. By following these steps, you'll have a working system that can handle appointment scheduling through AI voice calls.

Important Security Update: This application is designed as a single-user deployment where you host your own instance. All API keys and credentials are now stored securely in environment variables for enhanced security.

1

Fork the GitHub Repository

First, you'll need to create your own copy of the Calendly-Elevenlabs Integration repository:

  1. Navigate to the GitHub repository
  2. Click the "Fork" button in the top-right corner
  3. Select your GitHub account as the destination for the fork
  4. Wait for GitHub to create a copy of the repository under your account

Forking creates your own copy of the repository that you can deploy

Why Fork? Forking the repository allows you to have your own copy that you can customize and deploy. This is particularly important for the prompt builder feature, which needs access to your GitHub repository.
2

Deploy to Render

Render makes it easy to deploy your server with a single click:

  1. Navigate to your forked repository on GitHub
  2. Click the "Deploy to Render" button in the README.md file
  3. If you're not already logged in to Render, you'll be prompted to log in
  4. On the Render deployment page, configure the following settings:
    • Name: Choose a name for your service (e.g., "calendly-elevenlabs-server")
    • Region: Choose a region close to your location
    • Branch: main (or master, depending on your repository)
    • Root Directory: Leave blank
    • Environment: Node
    • Build Command: npm install
    • Start Command: node server.js
  5. Click "Create Web Service"
  6. Wait for Render to build and deploy your application (this may take a few minutes)

The Render deployment interface with recommended settings

Tip: You can also deploy manually by connecting your GitHub account to Render and selecting your forked repository.
3

Configure Environment Variables

After deployment, you need to configure your environment variables for security and functionality:

  1. In the Render dashboard, select your newly created service
  2. Go to the "Environment" tab
  3. Add the following environment variables:
Variable Name Description Required?
API_KEY A secret key for your server API (create your own secure string) Yes
ADMIN_PASSWORD Password for accessing admin pages (agent-builder, prompt-builder) Yes
CALENDLY_API_TOKEN Your Calendly API key Yes
ELEVENLABS_API_KEY Your Elevenlabs API key Yes
GITHUB_TOKEN Personal access token for your GitHub account (with repo scope) Yes
GITHUB_REPO_OWNER Your GitHub username Yes
GITHUB_REPO_NAME Name of your forked repository Yes
DEFAULT_TIMEZONE Default timezone (e.g., 'America/New_York') No
ENABLE_SMS Set to "true" to enable SMS confirmations Yes
TWILIO_ACCOUNT_SID Your Twilio account SID Yes
TWILIO_AUTH_TOKEN Your Twilio authentication token Yes
TWILIO_PHONE_NUMBER Your Twilio phone number Yes
Important: After adding environment variables, click the "Save Changes" button and then restart your service using the "Manual Deploy" button and selecting "Clear build cache & deploy".
Enhanced Security: API keys are now stored securely in environment variables instead of being entered in the user interface. This provides better security for your credentials.
Need help with API keys? Check the Integration Setup Guide for detailed instructions on obtaining API keys from each service.
4

Test Your Deployment

Now it's time to make sure everything is working:

  1. Go to your deployed application URL (e.g., https://your-app-name.onrender.com)
  2. You should see the home page with server status
  3. If the status shows "Server Online," your deployment is successful!
  4. Click on "Agent Builder" to start creating your voice agent
  5. You'll be prompted for authentication - enter any username (e.g., "admin") and the password you set in the ADMIN_PASSWORD environment variable

The home page should show "Server Online" if deployment is successful

Authentication: Admin pages (agent-builder.html, prompt-builder.html) are now protected with Basic Authentication. Use any username with the password you set in the ADMIN_PASSWORD environment variable.
Troubleshooting: If your server is not running or shows errors, check the logs in the Render dashboard. Common issues include missing environment variables or incorrect API keys.
